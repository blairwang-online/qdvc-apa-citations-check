# Technical info provided by Claude

**check_refs — APA Citation Checker** 

Audits a research paper (plain text) for APA citation consistency:
1. Inline citations with no matching reference-list entry
2. Reference-list entries with no matching inline citation
3. Inline citations that don't follow the configured comma style
4. Reference-list entries that aren't in alphabetical order
5. Inline citations using the wrong author separator ("and" vs "&")
6. Reference-list entries using the wrong author separator ("and" vs "&")
7. Reference-list entries using the wrong journal volume/issue format
8. Reference-list entries whose title isn't wrapped in the configured symbols

## File Overview

### `check_refs.py` (main file)

| Function | Description | Example Input | Example Output |
|---|---|---|---|
| `load_config(config_path=CONFIG_PATH)` | Reads style settings from a YAML config file, falling back to `DEFAULT_CONFIG` for any missing key or unreadable file. Returns a dict with all `DEFAULT_CONFIG` keys present. | `"config.yml"` (containing `uses_comma_intext: true`) | `{"uses_comma_intext": True, "intext_author_separator": "and", "reflist_author_separator": "&", "reflist_volume_issue_format": "comma"}` |
| `check_citations(file_path, config=None)` | Runs the full APA citation audit on the given file and prints a report. Settings come from the given config dict, or from `config.yml` via `load_config()` if not provided. | `"paper.txt"` (a file containing body text with (Author, Year) citations and a References section) | *(printed)* the eight-section APA CITATION AUDIT REPORT |

Style settings live in **`config.yml`** (read by `load_config`), with defaults in `DEFAULT_CONFIG`:
- `uses_comma_intext` — in-text citation style: `true` for `(Smith, 2026)`, `false` for `(Smith 2026)`. Default `false`.
- `intext_author_separator` — author separator before the final author in an in-text citation: `"and"` for `(Smith and Jones 2026)` or `"&"` for `(Smith & Jones 2026)`. Default `"and"`.
- `reflist_author_separator` — the same, but for reference-list entries: `"and"` for `Smith A and Jones B (2026) ...` or `"&"` for `Smith A & Jones B (2026) ...`. Default `"&"`.
- `reflist_volume_issue_format` — journal volume/issue format: `"comma"` for `Journal of Things, 16(2)` or `"parenthetical"` for `Journal of Things (16:2)`. Default `"comma"`.
- `reflist_title_start_symbol` / `reflist_title_end_symbol` — the symbols a reference title should start and end with, e.g. `"“"` / `"”"` for `“A study”`, or `"„"` / `"“"` for an asymmetrical `„A study“`. Set both to `""` to disable the title-symbol check. Default `"“"` / `"”"` (curly double quotes).

`check_refs.py` also defines `whitelist` in code — raw inline citation strings in this set are ignored entirely (e.g. for non-citation bracketed content you don't want flagged).

## Maintenance Notes

- **The Python modules must stay in the same directory.** `check_refs.py` imports directly from `qdvcrc_parsing`, `qdvcrc_analysis`, and `qdvcrc_report` by module name (no package structure), so they need to be co-located to run. `config.yml` is also read from the script's own directory (via `CONFIG_PATH`), so it should sit alongside them too.
- **Style configuration lives in `config.yml`, loaded by `load_config` in `check_refs.py`.** The loaded values are passed into the helper functions as parameters (e.g. `find_style_violations(raw_inline_citations, uses_comma_intext, ...)`), never imported into a `qdvcrc_*` module — this avoids a circular import back into the main file. Missing keys (or a missing file) fall back to `DEFAULT_CONFIG`; reading the YAML requires PyYAML (see `requirements.txt`). `whitelist` remains a code-level set in `check_refs.py`. If you add new configuration, add it to both `config.yml` and `DEFAULT_CONFIG`, then thread it through as a parameter.
- **Citation/reference matching is name+year based, not exact-text based.** Both inline citations and reference entries are reduced to `(Author, Year)` key tuples before comparison, so the checker is tolerant of minor formatting differences but can also produce false matches if two different authors share a surname and publication year.
- **Alphabetical-order checking compares each entry only to the one immediately before it**, using the surname before the first comma as the sort key. It flags the entry that breaks the sequence, not every entry that's technically misplaced — so a single line moved out of place will produce one violation, but it won't independently verify the full list is sorted from scratch. The sort key is ASCII-folded via `fold_to_ascii`, so accented surnames (e.g. `Öqvist`, `Bjørnsson`) sort by their base letters rather than by raw code point. This folds away the distinction between accented and base letters; it does not implement locale-specific collation (e.g. Swedish, where `å`/`ä`/`ö` sort after `z`).
- **Year detection spans the 1600s through the 2000s.** Both `YEAR_PATTERN` (inline) and `REFERENCE_YEAR_PATTERN` (reference list) match four-digit years beginning `16`, `17`, `18`, `19`, or `20`, so historical works (e.g. `Hobbes T (1651) Leviathan ...`) are handled. Keep the two patterns' century ranges in sync if you extend them.
- **Appendix material is checked just like body text.** After an `Appendix` heading, lines stay in citation-scanning mode, so inline citations in an appendix are matched against the reference list the same way body-text citations are. The `text` and `appendix` sections share an explicit branch in `parse_document`; only the `references` section feeds the reference data. A reference list that appears *after* an appendix heading is still parsed correctly, because the section flips again when the `References` heading is reached.
- **Author surnames may contain accented letters.** Surname extraction uses `AUTHOR_NAME_PATTERN`, a Unicode-aware regex that accepts an uppercase initial (ASCII or accented, e.g. the `Ä` in `Mähring`) followed by further letters, including hyphenated parts (e.g. `Pries-Heje`). It still requires at least two letters, so single-letter initials (`M` in `Mähring M`) are not treated as surnames. If you need to support scripts beyond the Latin-1 accented range (`À-ÖØ-Þ`), widen the leading character class.
- **Narrative (in-prose) citations are supported.** A citation like `Liesenfeld and Dingemanse (2024)`, where the authors are in the running text and only the year is parenthesized, is detected by `extract_narrative_citations` and matched against the reference list the same way as a bracketed citation. The authors are read from the contiguous run of capitalized name tokens immediately before the year parenthesis, so lowercase lead-in words ("according to", "by") are not mistaken for authors. The trade-off: a capitalized common noun sitting directly before a year parenthesis (e.g. `the Treaty (2019)`) can be misread as an author. Narrative citations are also excluded from the comma-style check, since the comma convention only applies to fully parenthetical citations — `parse_document` returns them as a fifth value (a set) that `check_refs.py` threads into `find_style_violations`.
- **Author-separator checks are presence-based, not exhaustive.** Sections 5 and 6 flag a citation/entry only when it actually contains the *disallowed* separator token (`&` when the config wants `"and"`, or the whole word `and` when the config wants `"&"`). They do not require the configured separator to be present, so single-author and comma-only entries (which contain neither token) are never flagged. The word `and` is matched only as a whole word, so it won't fire inside surnames like `Anderson`. The reference-list check (`find_reference_separator_violations`) inspects only the author prefix via `get_reference_author_prefix`, so `and`/`&` in a title or publisher is ignored; the in-text check needs no such guard because inline citations contain no title text. Configuration lives in `intext_author_separator` and `reflist_author_separator` in `config.yml`; an unrecognized value is treated as a no-op (no violations).
- **Volume/issue checking is presence-based and post-year only.** `find_volume_issue_violations` looks only at the text after the year parenthesis (via `get_reference_post_year`), so the parenthesized year and the author region can't be mistaken for a volume/issue. It distinguishes the two formats by the colon: `parenthetical` is a single paren joining volume and issue (`(16:2)`), while `comma` is a bare volume followed by an issue paren with no colon (`16(2)`, `16 (2)`, or `, 16(2)`). A line is flagged only when it matches the *other* format and not the configured one, so book entries and other references with no recognizable volume/issue are never flagged. Additionally, in `parenthetical` format a stray comma immediately before the paren (`Journal, (16:2)`) is flagged via `COMMA_BEFORE_PARENTHETICAL_PATTERN`, since the `(16:2)` part alone is well-formed and would otherwise slip through; a comma elsewhere in the title is not affected because the pattern requires the comma to be directly adjacent to the paren. Configuration lives in `reflist_volume_issue_format` in `config.yml`; an unrecognized value is a no-op.
- **Title-symbol checking extracts the title and checks its exact edges, and only for articles/chapters.** `find_title_symbol_violations` uses `extract_reference_title` to isolate the first segment after the year (stripping the leading `". "`/space) and flags it unless it both starts with `reflist_title_start_symbol` and ends with `reflist_title_end_symbol`. Extraction is symbol-aware: if the title opens with the start symbol, it runs through the matching end symbol, so a period inside the closing symbol (`"A study."`) doesn't truncate it. Setting either symbol to `""` disables the check (the "no symbols" style; section 8 then prints "Skipped"). Whole books are skipped via `is_article_or_chapter`, which treats an entry as an article/chapter only when it has a container marker (a journal volume/issue or an "In:" lead-in); this avoids false-flagging book titles, which aren't symbol-wrapped in these styles. The trade-offs: (1) an article that has neither a volume/issue nor an "In:" marker reads as a book and won't be checked (the safe direction); (2) a title with internal sentence-ending periods or an unusual journal boundary may be extracted imperfectly, which can cause a false flag.
